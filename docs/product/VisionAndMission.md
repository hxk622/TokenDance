# TokenDance 产品愿景与使命

**文档版本**: v2.0.0  
**更新日期**: 2026-01-14  
**核心理念**: Vibe-Agentic Workflow

---

## 💫 核心宣言

**Manus + Coworker + Vibe Workflow**

我们不是在构建另一个 Agent 工具，而是在创造一个将顶级 Agent 的"原子能力"通过"Vibe Workflow"进行重组和升华的**终极创作场**。

---

## 🎯 产品愿景 (Vision)

### 成为人机共生的"第一操作系统"

我们不只是一个应用，而是一个让顶级 Agent（**Manus 的执行力** + **Coworker 的本地深度**）在一种极具氛围感（**Vibe**）的环境中与人类意志无缝同频的空间。

**关键词**：人机共生 | 操作系统级 | 氛围感空间 | 意志同频

---

## 🚀 产品使命 (Mission)

### 将复杂的 Agent 智力，转化为丝滑的创作愉悦感

通过 **Vibe Workflow**，把 Coworker 对本地文件的极致操控力和 Manus 的全自动任务处理能力，封装成一种"所思即所得"的直觉化体验，让"Claude Code"这种硬核能力真正服务于全世界（**For the rest of the world**）。

**核心转化**：
- 技术复杂度 → 直觉化操作
- 命令行冰冷感 → 氛围感体验
- 多步骤交互 → 一键直达
- 开发者专属 → 全世界可用

---

## 🔱 三位一体核心架构 (The Trinity)

我们可以把这三者类比为：**大脑、双手与呼吸**。

| 维度 | 组件 | 角色定位 | 核心价值 |
|------|------|----------|----------|
| **执行大脑**<br>(The Brain) | **Manus** | 通用任务专家 | 负责高难度的外部决策、跨平台调度、以及从 0 到 1 的全自动交付。 |
| **执行双手**<br>(The Hands) | **Coworker** | 本地文件专家 | 继承 Claude Code 的基因，深度解析本地文件系统，实现精准的"外科手术式"代码/文档修改。 |
| **生命气息**<br>(The Vibe) | **Vibe Workflow** | 交互与情绪引擎 | 负责将前两者的强大能力化骨绵掌，通过 UI/UX 创造沉浸感，让复杂的协作变得优雅。 |

### 协同机制

```
用户意图 (模糊)
    ↓
[Vibe Workflow] → 意图捕捉、氛围营造
    ↓
┌─────────────────┐
│  路由决策层      │
└─────┬─────┬─────┘
      ↓     ↓
  [Manus] [Coworker]
   外部任务  本地文件
      ↓     ↓
  实时进度可视化
      ↓
  用户微调/确认
      ↓
  完美交付
```

## 🌊 核心理念：Vibe-Agentic Workflow

在这个愿景下，我们的产品设计遵循以下三个"超级理念"：

### A. 零阻力本地共生 (Frictionless Local Symbiosis)

**理念**  
借鉴 Coworker 对本地文件的友好性，但通过 Vibe UI 消除终端（Terminal）的冰冷感。

**对标与超越**  
- **Coworker 解决了**："能改本地文件"的问题
- **我们解决**："如何在改本地文件时，让用户感觉像在指挥一支交响乐团"的问题

**落地策略**
- 拖拽文件即可启动工作流
- 文件系统操作实时可视化
- 毛玻璃特效 + 丝滑动画反馈
- 沙盒安全 + 透明权限管理

---

### B. 异步与同步的完美流转 (Sync/Async Flow)

**理念**  
- **Manus** 擅长长时间任务（异步）
- **Coworker** 擅长即时修改（同步）
- **Vibe Workflow** 的核心在于管理这两者的"节奏"

**落地策略**

**当 Manus 在后台"攻城略地"时**：
- UI 呈现安静的进程美学
- 渐进式披露执行细节
- 可随时查看 Working Memory

**当 Coworker 在本地"精雕细琢"时**：
- 提供实时的、可感知的微动效反馈
- 文件变更即时高亮
- 支持中途调整和回滚

**节奏控制**：
```
长任务：[====进度条====] 60% (Manus 深度研究中...)
         ↓ 切换
短任务：[闪电图标] 文件已更新 (Coworker 完成 3 处修改)
```

---

### C. 意图的"模糊捕捉"与"精确映射"

**理念**  
针对"Rest of the world"（非技术用户），通过氛围感交互捕捉用户的模糊意图。

**落地策略**

**输入方式**：
- 拖拽文件 → Coworker 自动接入
- 点击意图卡片 → Manus 启动任务
- 自然语言 → Vibe 引导精炼

**Prompt 简化**：
- ❌ 不需要：写复杂的 Prompt
- ✅ 只需要：
  - 拖入文件夹
  - 选择意图模板（"Deep Research" / "PPT Generation" / "Code Review"）
  - 点击 "开始 Flow"

**Context 自动感知**：
```
用户操作：拖入 /project 文件夹
    ↓
Agent 立即完成：
  ✓ 文件树索引
  ✓ 代码语言检测
  ✓ 依赖关系分析
    ↓
UI 提示（丝滑动画）：
  "你的同事已就位，准备好开始 Flow 了吗？ 🚀"
```

## 📋 产品形态定义

### 三大核心模块

#### 1. Agent 协作中枢（Manus 与 Coworker 如何协同工作）

**场景 1：Deep Research（Manus 主导）**
```
用户意图："研究 Rust 异步编程最佳实践"
    ↓
Manus 启动：
  - Web 搜索 + 论文爬取
  - 多源信息整合
  - 生成结构化报告
    ↓
Coworker 辅助：
  - 将报告保存到本地 Workspace
  - 自动创建 Markdown + 参考文献
  - 更新项目文档索引
```

**场景 2：本地项目重构（Coworker 主导）**
```
用户意图："重构 /src 目录，使用组合模式替代继承"
    ↓
Coworker 启动：
  - 分析代码结构
  - 执行精确的文件修改
  - 保留原有代码（支持回滚）
    ↓
Manus 辅助：
  - 查找组合模式最佳实践
  - 生成测试用例
  - 更新架构文档
```

#### 2. 本地文件感知层（继承 Coworker 基因）

**核心能力**：
- 文件系统实时索引（类似 Spotlight）
- 语义化文件内容理解（向量化 + 图数据库）
- 支持拖拽、右键菜单快捷操作
- 沙盒权限管理（只能访问授权目录）

**安全设计**：
- 虚拟机隔离（参考 Anthropic 方案）
- 操作可审计（Context Graph 记录）
- 敏感文件保护（.gitignore 规则）

#### 3. 全局搜索与意图理解（产品核心竞争力）

**传统 Agent 的问题**：
- 需要写复杂 Prompt
- 文件上下文需要手动粘贴
- 多轮对话容易 Context Drift

**我们的解决方案**：
```
用户动作：拖入 /project 文件夹 + 点击 "Deep Research"
    ↓
系统自动完成（无需额外 Prompt）：
  ✓ 文件树解析（.md / .py / .ts）
  ✓ 依赖关系图谱
  ✓ 最近修改记录（Git）
  ✓ 相关文档检索
    ↓
Agent 获得完整 Context，开始执行
```

## 🎯 目标用户画像

### Persona 1: 技术创作者（早期核心用户）

- **背景**：独立开发者、技术作家、开源贡献者
- **痛点**：需要频繁切换 IDE、终端、浏览器、文档工具
- **期望**：一个能"读懂"本地项目、自动完成重复性工作的 AI 助手
- **使用场景**：
  - 代码审查 + 自动重构
  - 技术博客生成（从代码到文章）
  - 开源项目文档自动化

### Persona 2: 知识工作者（扩展用户）

- **背景**：研究员、咨询顾问、内容创作者
- **痛点**：信息过载、文档混乱、创作效率低
- **期望**：一个能"整理思路"、自动生成结构化输出的 AI 伙伴
- **使用场景**：
  - Deep Research → PPT 一键生成
  - 笔记整理 + 知识图谱构建
  - 多源信息聚合分析

### Persona 3: 企业团队（商业化方向）

- **背景**：产品团队、营销团队、客户成功团队
- **痛点**：跨部门协作低效、知识孤岛严重
- **期望**：一个能"承接任务"、自动生成交付物的 AI 团队成员
- **使用场景**：
  - 竞品分析报告自动化
  - 客户需求文档生成
  - 会议记录 + 行动项提取

## 🚨 风险与挑战

### 技术风险
1. **本地文件安全**：沙盒逃逸、提示注入攻击
   - 缓解：虚拟机隔离 + 操作审计 + 用户授权确认
2. **Agent 协同复杂度**：Manus 与 Coworker 的路由决策
   - 缓解：明确职责边界 + 智能路由算法 + 降级方案
3. **Context 管理**：大规模文件索引的性能问题
   - 缓解：增量索引 + 向量化检索 + 三文件工作法

### 产品风险
1. **用户认知门槛**：理解"Vibe-Agentic Workflow"需要教育
   - 缓解：强化 Onboarding + 案例展示 + 模板预设
2. **竞争压力**：Anthropic、OpenAI 快速迭代
   - 缓解：专注差异化（情感价值 + 开放平台）
3. **商业化路径**：免费 vs 付费的平衡
   - 缓解：Freemium 模型 + 企业版差异化
