# TokenDance 设计原则

**版本**: v2.0.0
**更新日期**: 2026-01-17
**整合来源**: UI-Design-Principles.md, Frontend-Design-Principles.md

---

## 核心定位

> TokenDance 不是 "更快的 Manus"，而是 **"可指挥的智能工作台"**。

用户是「导演」而非「观众」，强调「和 Agent 一起完成」而非「让 Agent 帮你」。

---

## 1. 三大核心原则

### 1.1 Transparency（透明度）

**核心理念**：用户能理解 Agent **为什么**做这个决策

**实现方式**：
- 推理链可视化（Chain-of-Thought）
- 决策热力图（Logits Heatmap）
- 执行流时间轴（可回放）
- 关键决策点高亮

**设计表现**：
```
✅ Agent 思考过程实时展示
✅ 工具调用详细日志
✅ 搜索/分析结果来源标注
✅ 错误时显示诊断信息
❌ 隐藏内部推理细节
❌ 直接返回结果（不展示过程）
```

---

### 1.2 Controllability（可控性）

**核心理念**：用户能在任何时刻干预和调整

**实现方式**：
- 人机接管点（中断/恢复）
- 动态方案选择（平行宇宙分支）
- 实时 Logits 调整（覆盖 AI 决策）
- 随时停止/回滚

**设计表现**：
```
✅ Agent 遇到障碍时显示 "🤚 接管" 按钮
✅ 多个方案对比（耗时/精确度权衡）
✅ 用户操作无缝衔接（从中断点恢复）
✅ 失败时提供替代路径
❌ 执行锁定（无法中断）
❌ 只有一个执行方案
```

---

### 1.3 Persistence（沉淀感）

**核心理念**：成功经验能被团队复用

**实现方式**：
- KV-Cache Snapshot（思维快照）
- 活的资产（Contextual Hot-link）
- Expert Agent 技能导出
- Workspace 团队共享

**设计表现**：
```
✅ 完整任务记录（生成于第 X 步）
✅ 关键产出物可标记为 "技能"
✅ 失败的分支也被保留（作为学习资源）
✅ 团队成员可一键复用成功方案
❌ 完成后直接删除
❌ 只保留最终结果
```

## 2. 避免 AI 味

### 2.1 视觉陈词滥调

| 类别 | ❌ 禁止 | ✅ 替代 |
|------|---------|---------|
| **字体** | Inter, Roboto, Arial | Space Grotesk（标题）+ 系统字体（正文） |
| **配色** | 紫色渐变 + 白背景 | 灰度 + 霓虹强调 |
| **图标** | Emoji (🎨 🚀 ⚙️) | Lucide Icons / Heroicons SVG |
| **布局** | 卡片 + 卡片 + 卡片 | 不对称、重叠、打破网格 |
| **动画** | 所有元素 scale(1.05) | 精心编排的交错显示 |

### 2.2 文案陈词滥调

| ❌ AI 味 | ✅ 人性化 |
|---------|-----------|
| "我能帮你..." | "你的智能工作台" |
| "让 AI 帮你..." | "和 Agent 一起完成任务" |
| "AI 助手" | "执行大脑 Manus" |
| "生成" / "创建" | "撰写" / "研究" / "整理" |

### 2.3 首页设计规范（AnyGen 风格）

**布局结构**（中央聚焦式）：
```
┌──────────────────────────────────────┐
│          Hero 标题 (居中)             │
│     "我能帮你做什么？"               │
├──────────────────────────────────────┤
│   ┌─────────────────────────────┐     │
│   │ 输入框 (textarea)           │     │
│   ├─────────────────────────────┤     │
│   │ [+] [人] ... [麦] [→]       │     │
│   └─────────────────────────────┘     │
├──────────────────────────────────────┤
│     快捷芯片 (chips) - 居中排列       │
│  [演示] [文档] [调研] [分析] ...    │
├──────────────────────────────────────┤
│     分类标签页 (tabs)                 │
│  [全部] [热门🔥] [市场] [产品] ...     │
├──────────────────────────────────────┤
│     模板卡片网格 (grid)              │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐           │
│  │PPT│ │调研│ │OKR│ │...│           │
│  └───┘ └───┘ └───┘ └───┘           │
└──────────────────────────────────────┘
```

**视觉风格**（浅色清爷）：
- **背景**：`#fafafa` + 点阵纹理
- **输入框**：白色卡片 + 圆角 16px + 微影
- **芯片**：白色背景 + 边框 + 圆角 9999px
- **模板卡片**：白色背景 + 彩色图标 + hover 抬起

**关键尺寸**：
- 内容最大宽度：`max-w-4xl` (896px)
- 顶部留白：`12vh`
- 输入框内边距：`16px 20px`
- 卡片网格：`2-4` 列响应式

**禁止**：
```
❌ 彩虹渐变背景
❌ 大面积毛玻璃
❌ Emoji 作为 UI 图标
❌ AI 助手开场白
✅ 克制的灰度系统
✅ Lucide 图标
✅ 任务导向文案
```

## 3. 设计思维流程

### 3.1 开始 UI 工作前的三个问题

| 问题 | 目的 | 示例 |
|------|------|------|
| **这解决什么问题？** | 理解上下文 | "执行页需要让用户感觉可控，而非被动等待" |
| **用户会记住什么？** | 定义差异化 | "三位一体的色球动画" / "能量连线流光效果" |
| **审美基调是什么？** | 承诺风格方向 | "工业实用 + 赛博朋克" |

### 3.2 TokenDance 审美定位

**当前选择**：**深色主题优先 + 工业赛博朋克元素**

**深色主题（默认）**：
- 主背景：`#0a0a0b`
- 卡片/侧边栏：`#141415`
- 悬浮/选中：`#1c1c1e`
- 霓虹强调色（青色 #00D9FF / 绿色 #00FF88 / 琥珀 #FFB800）
- 微妙边框：`rgba(255, 255, 255, 0.08)`
- 玻璃态背景：`backdrop-filter: blur(20px) saturate(180%)`

**浅色主题（备用）**：
- 灰度为主（#fafafa / #f1f5f9 背景）
- 黑色强调（#111827 / #1f2937）
- 细边框（1px gray-100/200）

**通用**：
- 圆角适中（8px / 12px / 16px）
- 动画时长：150-300ms

## 4. 交互信条

- ❌ 不要让用户 "等着 AI 完成"
- ✅ 让用户 "和 AI 一起完成"

- ❌ 不要让失败的任务白白浪费
- ✅ 让失败的尝试变成可调试的录像

- ❌ 不要让专家经验只存在个人脑海
- ✅ 让专家经验沉淀为团队资产

## 6. 去 AI 味检查清单

### 视觉
- [ ] 无 Emoji 图标
- [ ] 图标来自 Lucide/Heroicons
- [ ] 配色非紫色渐变
- [ ] 无 "AI 助手" 类开场白

### 交互
- [ ] 所有可点击元素有 `cursor-pointer`
- [ ] hover 不导致布局偏移
- [ ] 过渡时间 150-300ms
- [ ] 关键动画有交错 `animation-delay`

### 文案
- [ ] 无 "AI 助手" 类表述
- [ ] 强调用户主动性（"接管" / "干预"）
- [ ] 任务导向（"撰写报告" vs "生成报告"）

---

## 7. 审美优化检查清单

### 深色主题层次
- [ ] 背景使用 `#0a0a0b` 而非纯黑
- [ ] 卡片有微妙亮度差异（`#141415`）
- [ ] 玻璃态元素有 `backdrop-filter: blur()`
- [ ] 边框使用 `rgba(255, 255, 255, 0.08)` 而非纯灰

### 动画与过渡
- [ ] 页面入场有交错动画（stagger）
- [ ] CTA 按钮有 hover surprise 效果
- [ ] 加载/执行状态有脉冲动画
- [ ] 进度条有流光效果（shimmer）

### 响应式
- [ ] Header 在移动端有合理折行
- [ ] 卡片网格在窄屏变为单列
- [ ] 模态框在小屏全宽显示

### 微交互
- [ ] 按钮有 active 按压反馈（scale 0.97-0.98）
- [ ] 卡片有光晕/聘光灯效果
- [ ] 输入框 focus 有渐变边框
- [ ] 列表项 hover 有侧边指示条
